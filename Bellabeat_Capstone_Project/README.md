# Bellabeat Data Analytics Case Study

## Project Overview
This case study aims to analyze smart device usage data to gain insights into consumer behavior and preferences, guiding Bellabeat's marketing strategy to empower women with knowledge about their health and habits.

## Business Task
- **Objective:** Analyze smart device usage data to understand how consumers use non-Bellabeat smart devices and apply these insights to Bellabeat's product marketing strategy.
- **Key Questions:**
  1. What are some trends in smart device usage?
  2. How could these trends apply to Bellabeat customers?
  3. How could these trends help influence Bellabeat's marketing strategy?

## Data
The dataset consists of personal tracker data from thirty eligible Fitbit users, including minute-level outputs for physical activity, heart rate, and sleep monitoring, collected over a month-long period.

## Methodology
1. **Prepare and Process:**
   - Data storage in SQL and organization in long and wide formats.
   - Data cleaning and removal of users with insufficient data.

2. **Analyze:**
   - Identified trends and relationships between various metrics.

3. **Share:**
   - Key insights and visualizations.
   - Recommendations for product development and marketing strategy.

## How to Run the Analysis
1. Clone the repository.
2. Navigate to the `sql_queries` directory and run the SQL scripts to clean and prepare the data.
3. Open the Jupyter notebook in the `notebooks` directory to explore the data and generate visualizations.
4. Review the `report` directory for detailed insights and recommendations.
